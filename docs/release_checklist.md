# Release Checklist

- [ ] Confirm the software licence and third-party code notices.
- [ ] Remove local absolute paths from examples and documentation.
- [ ] Verify that no trained checkpoint or large raster product is committed to Git.
- [ ] Run syntax checks for all Python files.
- [ ] Run a small train/inference/validation smoke test.
- [ ] Create a GitHub release tag.
- [ ] Archive the code release and obtain a software DOI.
- [ ] Add reciprocal links between the code release and the data repository.
